# E-Devlet Official Evidence Policy

## Scope
This policy defines the legitimate handling model for e-Devlet, UCBP/TUCBS, TKGM, and user-provided official parcel evidence within ParselRadar.

## Absolute Security Rules
- ParselRadar never asks for e-Devlet password.
- ParselRadar never stores T.C. Kimlik number.
- ParselRadar never stores e-Devlet session cookies.
- ParselRadar never automates e-Devlet login.
- ParselRadar never asks users to share SMS or MFA codes.
- ParselRadar never scrapes authenticated state portals.
- ParselRadar never impersonates a user.

## Allowed Use
- User may manually log in to e-Devlet, UCBP, or TKGM outside ParselRadar.
- User may download official documents or screenshots.
- User may upload those documents or screenshots to ParselRadar.
- Uploaded files are treated as user-provided evidence.
- ParselRadar labels evidence source clearly in outputs and reports.

## Forbidden Use
- Collecting e-Devlet credentials.
- Automated login.
- Session replay.
- Shared user accounts.
- Hidden scraping.
- Storing sensitive personal identity data unnecessarily.

## User-Facing Principle
ParselRadar provides preliminary real estate intelligence and evidence organization, not official government certification.

## Evidence Source Taxonomy
ParselRadar report labels must distinguish:
- VERIFIED_BY_APPROVED_CONNECTOR
- USER_PROVIDED_OFFICIAL_EVIDENCE
- PUBLIC_SOURCE
- UNVERIFIED
- MISSING
- NEEDS_MANUAL_REVIEW
