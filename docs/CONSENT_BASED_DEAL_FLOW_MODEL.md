# CONSENT-BASED DEAL FLOW MODEL

## Purpose
Define how ParselRadar can support lead and deal intelligence while protecting user control, privacy, and trust.

## Opt-In Model
Core rule:
- No sharing for deal-flow purposes without explicit user opt-in.

Minimum opt-in characteristics:
- Clear and separate from general terms acceptance
- Context-specific (what is shared, with whom, and why)
- Revocable by user
- Logged for auditability

## User Consent Language Principles
Consent language should be:
- Plain and understandable
- Specific about data categories
- Specific about recipient categories
- Specific about expected outcome
- Explicit about revocation rights

## What Data Can Be Shared
Only when consent exists:
- Selected property metadata
- Selected evidence summaries
- Readiness and missing-check summaries
- Contact channel preferences

## What Data Must Stay Private
Unless separately and explicitly consented:
- Sensitive account details
- Private notes not marked for sharing
- Full document payloads by default
- Internal risk labels intended for internal review only

## Anonymized Lead Model
Optional pathway:
- Share anonymized opportunity signals first
- Reveal identity/contact only after explicit permission
- Use minimum necessary data at each stage

## Professional Marketplace Model
Allowed model:
- Consent-based matching between users and professionals
- Permissioned access to relevant case context
- Structured acceptance flow for professional engagement

Not allowed model:
- Automatic hidden lead routing
- Forced contact transfer without user action

## Operator/Admin Boundaries
Operator/admin responsibilities:
- Enforce consent checks before sharing
- Maintain logs for consent and sharing actions
- Apply policy guardrails consistently

Operator/admin limits:
- No bypass of user consent controls
- No off-platform hidden sharing of user data
- No unauthorized profiling for deal distribution

## Prohibited Hidden Data Use
Prohibited actions:
- Silent lead resale
- Sharing identifiable user/property data without consent
- Backdoor access for preferred professionals
- Reusing private data for unrelated commercial targeting

## Future Dashboard Implications
User workspace implications:
- Clear consent settings panel
- Share status visibility per property/case
- Easy revoke and history view

Professional workspace implications:
- Permission-gated lead inbox
- Visibility only into consented data
- Clear scope boundaries for each lead/case

Admin workspace implications:
- Consent audit timeline
- Policy compliance alerts
- Controlled override mechanics (if any) with strict audit trails

## Strategic Guardrail
Consent-based deal-flow is a trust feature, not a hidden marketplace shortcut.
ParselRadar remains an evidence and pre-check intelligence platform, and all deal-flow actions must remain user-permissioned.
