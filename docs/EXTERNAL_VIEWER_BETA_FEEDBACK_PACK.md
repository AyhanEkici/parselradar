# EXTERNAL VIEWER BETA FEEDBACK PACK

## 1) Short Intro
Thank you for joining the ParselRadar controlled-beta walkthrough.

ParselRadar is a controlled-beta intelligence workflow that helps structure parcel evidence and risk signals into a more disciplined, repeatable review process.

This demo is guidance-oriented. It does not provide official verification, official valuation, legal determination, or automated government decisions.

## 3) Feedback Questions
### Clarity
- Which screen was most clear?
- Which screen was hardest to understand?
- Which term or phrase felt too technical?

### Usefulness
- Which part would help you most in a real parcel review?
- Which missing-evidence prompt felt most actionable?
- What feature or wording would improve practical usefulness immediately?

### Trust
- Did any part sound like an overclaim?
- Did you clearly understand this is guidance-only?
- Did you clearly understand official checks remain manual?

### Confidence
- Would you use this in a controlled-beta review workflow today?
- What one change would most increase your confidence?

## 4) Safe Limitation Wording
Use this wording during and after the session:

- This is a controlled-beta workflow, not public launch.
- The report is guidance, not official verification.
- Official checks remain manual unless permissioned integrations are approved.
- TKGM and e-Imar are not automated in this flow.
- Manual evidence is still required.
- Some metadata completeness is still being hardened before public launch.
- Mobile admin experience is acceptable for controlled beta but not final.
- First successful GitHub Actions scheduled run is still not verified.

## 5) Do-Not-Say Reminders
- Officially verified
- Guaranteed valuation
- Buy this property

- Public launch ready

## 7) Viewer #2 Structured Feedback Mode
- Do not accept only "everything is good" as sufficient feedback.
- Ask these five required questions in every Viewer #2 session:
	- What do you think ParselRadar does?
	- Where would you use this?
	- What is unclear or too limited?
	- Do you trust the report as controlled-beta guidance, not official verification?
	- Would you test this with a real parcel or pay for it later?
- Record concrete confusion, value, and trust signals.
- Keep public launch status as NOT_READY during all messaging.
