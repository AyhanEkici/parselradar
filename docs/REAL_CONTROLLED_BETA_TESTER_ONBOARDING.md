# REAL CONTROLLED BETA TESTER ONBOARDING

## Phase
- Phase: P2.BETA-PILOT-2
- Scope: Controlled beta operations only
- Public launch status: NOT_READY
- Controlled beta status: READY_FOR_CONTROLLED_BETA

## 1) Pilot Objective
- Validate real user understanding of product scope and boundaries.
- Validate property submission flow completion.
- Validate document upload and missing-evidence guidance flow.
- Validate report usefulness and readability for next-step decision support.
- Validate admin review visibility across CMS, analysis reports, and deal-flow.
- Validate mobile friction points in real usage.
- Validate no misleading claims are shown in user/admin surfaces.

## 2) Tester Profile
- Invite 1-3 known real testers only.
- Include buyer/investor profile testers.
- Include professional/agent/admin-adjacent perspective if available.
- No public launch communication.
- No paid campaign or open enrollment.

## 3) Tester Prerequisites
- Test account is created and reachable.
- Tester acknowledges this is controlled beta.
- Tester acknowledges reports are informational pre-checks only.
- Tester acknowledges no official TKGM/municipality/tapu/legal/valuation replacement is claimed.

## 4) Exact Tester Tasks
1. Open homepage and read platform boundary text.
2. Register/login.
3. Create a new property or inspect an existing test property.
4. Upload or simulate supporting documents.
5. Review missing evidence guidance.
6. Open result/report page.
7. Read Evidence Matrix.
8. Read Source Guidance Summary.
9. Open credits page.
10. Test same flow on mobile viewport.
11. Report confusion points, blockers, and unclear language.

## 5) Admin Tasks
1. Review tester-created/updated property records.
2. Open admin CMS.
3. Open admin analysis reports.
4. Open admin deal-flow.
5. Classify issues by P0/P1/P2/P3.
6. Keep external sharing disabled.
7. Do not activate buyer/professional matching unless explicit consent and manual review exist.

## 6) Pass/Fail Criteria
Pass criteria:
- No unresolved P0/P1 blocker.
- Tester can complete core flow end-to-end.
- No fake official-proof, valuation, or buy-advice claim appears.
- Admin route separation remains intact.
- Report surface is understandable for controlled beta use.
- Public launch status remains NOT_READY.

Fail criteria:
- Any unresolved P0/P1 blocks core flow.
- Misleading official-proof/valuation/buy claim is found.
- Admin route separation fails.
- Public-launch-ready overclaim appears.

## 7) Pilot Execution Envelope
- Run with 1-3 known testers.
- Track each run in the beta issue ledger before expanding tester count.
- Expansion beyond 3 testers requires explicit review of P0/P1 issue state.

## 8) Tester Invitation Template (Optional)
Use this short message template for known testers:

The platform provides informational pre-check and evidence organization support only, and does not replace official/legal verification. Please complete the test flow (homepage, login, property, documents, report, credits, mobile) and send feedback on confusion points or blockers."
