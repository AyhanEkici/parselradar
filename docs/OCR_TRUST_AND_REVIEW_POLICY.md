# OCR TRUST AND REVIEW POLICY

## 1) OCR Output Is Never Official Proof
OCR-derived text or fields are supportive hints only and cannot be treated as legal/official proof.

## 2) OCR Output Must Be Review-Gated
Every OCR output must pass manual review before being considered for downstream usage.

## 3) OCR Output Must Display Confidence
Confidence must be visible and interpreted as probabilistic guidance, not certainty.

## 4) OCR Cannot Produce Buy/Sell Advice
OCR components must not provide investment, buy, or sell recommendations.

## 5) OCR Cannot Create Valuation
OCR components must not generate or imply official/final valuation outcomes.

## 6) OCR Cannot Claim TKGM/e-Imar Automation
OCR features must not claim active automation of TKGM, e-Imar, or official systems.

## 7) OCR Cannot Bypass Login/CAPTCHA/e-Devlet
No bypass logic, scraping, credential capture, or automation of protected portals is allowed.

## 8) OCR Should Preserve Original Uploaded Evidence
Original evidence artifacts must remain the primary reference; OCR is secondary interpretation.

## 9) OCR Should Require Manual Confirmation Before Report Use
No OCR-extracted field may affect report readiness or trust labels without explicit reviewer confirmation.

## 10) Audit Trail Requirements
Minimum audit trail:
- extraction timestamp
- model/tool identifier (if used later)
- field-level confidence
- reviewer decision (accept/reject)
- reviewer identity
- source artifact reference

## 11) User-Facing Wording Rules
Required wording patterns:
- "Guidance only"
- "Manual review required"
- "Not official verification"
- "May contain OCR errors"

Forbidden wording patterns:
- "Officially verified by OCR"
- "Legally confirmed"
- "Guaranteed accurate"

## 12) Do-Not-Say List
Do not say:
- "OCR confirms legal title"
- "OCR validates municipality approval"
- "OCR proves valuation"
- "OCR automates e-Devlet/TKGM/e-Imar access"
- "OCR recommendation: buy/sell"

## 13) Enforcement Baseline
- Trust warnings must remain visible wherever OCR output appears.
- Manual review gating must be non-optional.
- Any overclaiming behavior is a release blocker.
